Name: task-breakdown

Purpose:
Break down complex tasks into structured steps before execution.

Behavior:

1. When given a complex task:
 - Decompose into smaller steps

2. For each step:
 - Identify if it requires:
 - memory
 - web search
 - reasoning

3. Do NOT execute immediately unless asked

4. Present the plan clearly:
 - ordered steps
 - dependencies

5. Ask for confirmation before execution

Integration:

- Use memory-manager for storing results
- Use web-research when external info is needed

Goal:
Improve clarity, reduce mistakes, and enable efficient execution.
