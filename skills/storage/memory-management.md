# Memory Manager Skill

## Purpose
Efficiently store, update, and retrieve information for future use. Good memory management reduces costs and improves response relevance.

## Core Principles

### 1. What to Store
Only keep information that is:
- **Reusable** – can be applied in multiple contexts
- **Important** – critical for task completion or decision-making
- **Likely to be needed again** – future reference is probable

### 2. What to Avoid
Never store:
- Temporary answers (e.g., one-time calculations, ephemeral results)
- One-time tasks that won't recur
- Unverified or low-confidence information
- Duplicate entries across files

### 3. Storage Practice
Always:
- **Summarize before saving** – condense key points, discard noise
- **Use structured formats** – markdown, lists, tables when appropriate
- **Prefer updating existing memory** over creating new files
- **Check memory first** before performing a web search to avoid duplication
- **Save useful insights** after a successful search (with source citation)

### 4. File Organization
- Keep memory root clean; use subdirectories (`memory/user/`, `memory/knowledge/`, `memory/web/`).
- Each file should have a clear, descriptive purpose.
- When in doubt, consolidate rather than proliferate.

### 5. Review & Maintenance
Periodically review stored information for relevance; prune outdated or superseded facts.

## Usage
When instructed to remember or store something:
1. Evaluate against the "what to store" criteria.
2. Find the appropriate existing file; create a new one only if necessary.
3. Summarize and add in structured form; cite sources if derived from web.
4. If unsure where to place information, ask before writing.
